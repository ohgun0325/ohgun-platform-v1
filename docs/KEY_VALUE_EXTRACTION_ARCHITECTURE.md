## 🔄 데이터 흐름 상세

### Flow 1: Structured PDF 처리

```
PDF 파일
   │
   ├─> PyMuPDF.open(pdf_path)
   │
   ├─> page.get_text("words")
   │   │
   │   └─> [(x0, y0, x1, y1, "text", block, line, word), ...]
   │
   ├─> Word 객체 변환
   │   │
   │   └─> [Word(text, BBox(x0,y0,x1,y1), ...), ...]
   │
   ├─> 키워드 매칭
   │   │
   │   └─> key_words = [Word("성명", ...), ...]
   │
   ├─> 각 키워드별 후보 탐색
   │   │
   │   ├─> 거리 계산: distance = sqrt((x1-x2)² + (y1-y2)²)
   │   ├─> 방향 판단: if same_line → SAME_LINE, else → RIGHT/BELOW/...
   │   ├─> 필터링: 자기자신, 라벨, 최대거리 제외
   │   │
   │   └─> candidates = [{"word": ..., "direction": ..., "distance": ...}, ...]
   │
   ├─> 점수 계산 및 최적 선택
   │   │
   │   ├─> for each candidate:
   │   │     score = (direction_weight × alignment_bonus) / (distance + 1)
   │   │
   │   └─> best = max(candidates, key=score)
   │
   └─> KeyValuePair 생성
       │
       └─> {key, value, confidence, direction, distance, bbox}
```

### Flow 2: Scanned PDF 처리

```
PDF 파일
   │
   ├─> PyMuPDF.open(pdf_path)
   │
   ├─> page.get_pixmap(matrix=Matrix(2.0, 2.0))
   │   │
   │   └─> PIL Image (고해상도)
   │
   ├─> EasyOCR.readtext(image)
   │   │
   │   └─> [(bbox_4points, text, confidence), ...]
   │
   ├─> bbox 변환 (4점 → 직사각형)
   │   │
   │   ├─> x0 = min(point[0] for all 4 points)
   │   ├─> y0 = min(point[1] for all 4 points)
   │   ├─> x1 = max(point[0] for all 4 points)
   │   └─> y1 = max(point[1] for all 4 points)
   │
   ├─> Word 객체 변환
   │   │
   │   └─> [Word(text, BBox(x0,y0,x1,y1), ...), ...]
   │
   └─> KeyValueExtractor로 매칭
       (이후 Flow 1과 동일)
```

## 🎛️ 설정 매트릭스

### 문서 타입별 권장 설정

| 문서 타입 | max_distance | same_line_tolerance | 방향 가중치 | 비고 |
|-----------|--------------|---------------------|-------------|------|
| **표준 양식** (수평) | 300px | 5px | 기본값 | 가장 일반적 |
| **수직 양식** | 300px | 5px | BELOW: 8.0 | 아래 방향 강화 |
| **촘촘한 표** | 150px | 3px | 기본값 | 거리/허용오차 줄임 |
| **넓은 표** | 500px | 10px | 기본값 | 거리/허용오차 늘림 |
| **혼합형** | 300px | 7px | 기본값 | 약간 여유있게 |
| **영문 문서** | 300px | 5px | 기본값 | 키워드만 영문으로 |

### PDF 타입별 처리 전략

| PDF 타입 | 처리 방법 | 예상 시간 | GPU 필요 | 정확도 |
|----------|-----------|-----------|----------|--------|
| **Structured** (텍스트) | PyMuPDF 직접 | ~30ms | ❌ | 95%+ |
| **Scanned** (이미지) | PyMuPDF→OCR | ~500ms | ✅ 권장 | 85%+ |
| **Mixed** (혼합) | 페이지별 자동 선택 | 가변 | ✅ 권장 | 90%+ |
| **Hybrid** (일부 스캔) | 폴백 사용 | 가변 | ✅ 권장 | 90%+ |

## 🎯 알고리즘 의사결정 트리

### 방향 판단

```
Key와 Value bbox 주어짐
    │
    ├─> |key.y - value.y| <= tolerance?
    │       │
    │       ├─ Yes ─> SAME_LINE
    │       │           │
    │       │           └─> value.x > key.x? → SAME_LINE
    │       │               value.x < key.x? → LEFT
    │       │
    │       └─ No ─> h_dist vs v_dist 비교
    │                   │
    │                   ├─ h_dist > v_dist?
    │                   │   │
    │                   │   ├─ Yes ─> value.x > key.x? → RIGHT
    │                   │   │         value.x < key.x? → LEFT
    │                   │   │
    │                   │   └─ No ─> value.y > key.y? → BELOW
    │                   │             value.y < key.y? → ABOVE
    │                   │
    │                   └─> Direction 결정 완료
```

### 후보 필터링

```
모든 단어 순회
    │
    For each word:
    │
    ├─> word == key_word? → 제외 (자기 자신)
    │
    ├─> normalize(word) == normalize(key)? → 제외 (같은 텍스트)
    │
    ├─> distance > max_distance? → 제외 (너무 멀리)
    │
    ├─> word.text ends with [:：\(\[\{]? → 제외 (라벨 패턴)
    │
    ├─> len(word.text) < 1? → 제외 (너무 짧음)
    │
    └─> 통과 → 후보에 추가
```

## 📊 성능 비교표

### 처리 시간 분해

#### Structured PDF

| 단계 | 시간 | 비율 |
|------|------|------|
| PDF 열기 | ~5ms | 17% |
| get_text("words") | ~10ms | 33% |
| 키워드 매칭 | ~5ms | 17% |
| 후보 탐색 + 점수 | ~8ms | 27% |
| 결과 생성 | ~2ms | 6% |
| **총계** | **~30ms** | **100%** |

#### Scanned PDF (GPU)

| 단계 | 시간 | 비율 |
|------|------|------|
| PDF 열기 + 렌더링 | ~50ms | 10% |
| EasyOCR 실행 | ~400ms | 80% |
| bbox 변환 | ~10ms | 2% |
| 키워드 매칭 | ~5ms | 1% |
| 후보 탐색 + 점수 | ~15ms | 3% |
| 결과 생성 | ~20ms | 4% |
| **총계** | **~500ms** | **100%** |

**병목 지점**: EasyOCR (80%) → GPU 필수

## 🔧 튜닝 결정 트리

### "값이 추출 안 됨" 문제 해결

```
값이 추출 안 됨
    │
    ├─> 1. 키워드가 문서에 있는가?
    │       │
    │       ├─ 없음 → 키워드 변경 또는 추가
    │       │          (띄어쓰기 변형, 동의어)
    │       │
    │       └─ 있음 → 2로 이동
    │
    ├─> 2. PDF가 Structured인가?
    │       │
    │       ├─ Scanned → OCR 활성화
    │       │             ocr_reader 제공
    │       │
    │       └─ Structured → 3으로 이동
    │
    ├─> 3. 값이 너무 멀리 있는가?
    │       │
    │       ├─ Yes → max_distance 증가
    │       │         (300 → 500)
    │       │
    │       └─ No → 4로 이동
    │
    ├─> 4. 값이 비선호 방향인가?
    │       │
    │       ├─ Yes → 방향 가중치 조정
    │       │         (BELOW 또는 LEFT 증가)
    │       │
    │       └─ No → 5로 이동
    │
    └─> 5. 정렬이 안 되어 있는가?
            │
            ├─ Yes → same_line_tolerance 증가
            │         (5 → 10)
            │
            └─ No → 수동 디버깅 필요
                     (raw_words 확인)
```

### "잘못된 값 추출됨" 문제 해결

```
잘못된 값 추출됨
    │
    ├─> 1. 거리가 너무 먼가?
    │       │
    │       ├─ Yes → max_distance 감소
    │       │         (300 → 150)
    │       │
    │       └─ No → 2로 이동
    │
    ├─> 2. 다른 후보가 더 가까운가?
    │       │
    │       ├─ Yes → direction_weights 조정
    │       │         선호 방향 강화
    │       │
    │       └─ No → 3으로 이동
    │
    ├─> 3. 라벨이 값으로 인식되는가?
    │       │
    │       ├─ Yes → 키워드를 더 구체적으로
    │       │         post_process 검증 추가
    │       │
    │       └─ No → 4로 이동
    │
    └─> 4. 신뢰도 임계값 상향
            confidence >= 0.8
```

## 🎓 알고리즘 핵심 개념 요약

### 1. 좌표 시스템

```
PDF 좌표계 (PyMuPDF):
    
    (0,0) ────────────> X축
      │
      │
      │
      ▼
    Y축

단어 bbox:
    (x0, y0) ┌──────────┐
             │   Text   │
             └──────────┘ (x1, y1)

중심점:
    center_x = (x0 + x1) / 2
    center_y = (y0 + y1) / 2
```

### 2. 거리 메트릭

```
유클리드 거리:
    d = √[(x₂-x₁)² + (y₂-y₁)²]

맨해튼 거리:
    d = |x₂-x₁| + |y₂-y₁|

본 시스템: 유클리드 거리 + 방향 가중치
```

### 3. 점수 함수

```
기본 점수:
    score = weight / (distance + 1)

정렬 보너스 적용:
    score = (weight × bonus) / (distance + 1)
    
    bonus = 1.5 if aligned else 1.0

신뢰도 변환:
    confidence = sigmoid(score - 1)
               = 1 / (1 + e^(1-score))
```
